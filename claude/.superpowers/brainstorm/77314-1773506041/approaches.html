三种扩展方案

基于 everything-claude-code 底座，往上叠加能力的三条路径

A

MCP 优先

先打通外部工具连接，再做其他优化


第1步：装核心 MCP（GitHub、Playwright、Postgres、Notion、搜索）

第2步：更新 CLAUDE.md 让 agents 知道如何用这些工具

第3步：加 Trellis 做项目记忆


优势

  • 立竿见影，感受最明显
  • 解决"基础版"核心问题

代价

  • 部分 MCP 需要 API key
  • 配置量较大
B

全面重构

一次性把所有层都优化：MCP + UI风格 + CLAUDE.md + 自定义skills


第1步：重新设计 CLAUDE.md（输出风格、UI规范）

第2步：装所有 MCP servers

第3步：加 Trellis + 自定义 skills

第4步：针对你的工作场景定制 agents


优势

  • 最终效果最完整
  • 一步到位

代价

  • 复杂度高，容易出问题
  • 难以定位哪步出了问题
C

逐层叠加 ⭐ 推荐

分阶段验证，每层稳了再加下一层


第1层：激活已有 UI skills（frontend-design、ui-ux-pro-max）

第2层：装 MCP servers（按优先级分批）

第3层：Trellis + 跨项目记忆

第4层：CLAUDE.md 精调 + 自定义 skills


优势

  • 风险低，每步可验证
  • 出问题容易定位
  • 可以随时暂停

代价

  • 见到完整效果慢一些
